# OpenSCP Update Summary — 2026-03-06

## Structural Upgrade: Dynamic Theming Paradigm

Today, the application underwent a major architectural shift regarding UI styling. The "Hardcoded CSS" approach was abandoned in favor of a **Centralized Theme Engine**.

### Key Changes
1.  **CSS Decoupling**: Removed all `setStyleSheet` calls with hardcoded hex colors from:
    *   `MainWindow` (Toolbar buttons, status bar).
    *   `LocalPanel` & `RemotePanel` (Headers, trees, context menus).
    *   `BreadcrumbNavigator` (Path bars, segment buttons).
    *   `TasksPanelWidget` (Task list, status labels).
    *   `ConnectionManagerDialog` (Titles, separators).
2.  **Object Name Mapping**: Implemented `setObjectName()` across all key widgets. The `theme_manager.py` now uses these IDs (`btn_connections`, `local_panel_title`, etc.) to apply surgical styles via global QSS.
3.  **Light Theme Support**: 
    *   Updated `theme_to_qss` to explicitly define `selection-color` and `background` for all views, preventing OS-native color leakage.
    *   Added `GitHub Light` and `Solarized Light` themes.
4.  **Terminal Stability**:
    *   Implemented handling for `\r` (Carriage Return) to prevent prompt duplication.
    *   Added support for standard Unix shortcuts: `Ctrl+W` (delete word), `Ctrl+U` (delete line), `Alt+Backspace`.
    *   Fixed arrow key navigation to be non-destructive.
5.  **UI/UX Refinement**:
    *   `SettingsDialog`: Increased default size, fixed vertical overlap, and renamed "Cancel" to "Close" to reflect real-time saving.
    *   Standardized icons and spacing across dual-panels.

## New Theme Palette
*   **Dark Default (Neon Edition)**: Graphite grey base with vibrant Neon Green highlights.
*   **One Dark**: Professional blue-grey theme.
*   **GitHub Light**: High-contrast clean white theme.
*   **Solarized Light**: Cream-based ergonomic light theme.
*   **Monokai**: Classic neon-on-charcoal theme.

## Important for LLMs
**DO NOT add hardcoded CSS strings in Python files.** All new components must use `setObjectName` and have their styles defined in `openscp/utils/theme_manager.py` using theme tokens.
